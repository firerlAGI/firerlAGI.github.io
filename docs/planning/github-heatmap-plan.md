## GitHub 提交热力图实现计划

### 📋 功能概述
实现类似 GitHub 个人主页的提交热力图，显示过去一年（52周）的每日贡献情况，用颜色深浅表示贡献程度。

### 🎯 实现步骤

#### 阶段一：UI 设计（ui-designer 智能体）
1. 设计热力图视觉风格，确保与现有网站风格一致
2. 设计4级贡献颜色方案（无贡献、低、中、高）
3. 设计悬停提示（tooltip）的交互效果
4. 设计图例说明的布局
5. 设计移动端响应式方案（横向滚动）

#### 阶段二：前端实现（frontend-architect 智能体）
1. 扩展 HTML 结构
   - 在 `index.html` 的 `#github-section` 中添加热力图容器
   - 添加月份标签和星期标签
   - 添加图例说明容器

2. 扩展 CSS 样式
   - 定义热力图网格布局（7列 x 52行）
   - 实现基于 ui-designer 设计的颜色方案
   - 添加悬停提示样式
   - 实现响应式适配

3. 扩展 JavaScript 功能
   - 数据聚合：从 GitHub events 提取 PushEvent，按日期统计提交次数
   - 热力图生成：生成 52周 x 7天的网格结构
   - 颜色映射：根据提交次数映射到不同颜色等级
   - 交互功能：悬停显示具体日期和提交数
   - 空状态处理

### 📊 技术方案
- **数据源**：继续使用现有的 `fetchGitHubActivity()` 获取 events
- **时间范围**：显示最近 365 天的数据
- **贡献等级**：4级（0次、1-2次、3-5次、6次以上）
- **缓存策略**：复用现有缓存机制

### 🤖 执行智能体
1. **ui-designer** - 负责热力图的视觉设计、颜色方案、交互效果设计
2. **frontend-architect** - 负责实际的 HTML/CSS/JavaScript 组件实现
